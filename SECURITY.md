# Security Policy

## Supported Versions

The active default branch is supported for security updates.

## Reporting A Vulnerability

Please do not open public issues for vulnerabilities.

Report security concerns privately with:

- Affected component
- Reproduction details
- Impact summary
- Suggested mitigation if known

Use private maintainer contact channels described in SUPPORT.md.

## Security Practices In This Repository

- Secret values are expected through environment variables.
- Quality gates include secret leak checks for generated content.
- Verification and diagnostics should avoid leaking token values.

## Response Expectations

- Initial acknowledgement target: within 72 hours.
- Triage and severity classification: as soon as reproducible details are available.
- Public disclosure timing: coordinated with maintainers after mitigation.
