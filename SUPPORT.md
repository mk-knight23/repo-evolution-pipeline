# Support

## Getting Help

Use repository issues for:

- Bug reports
- Feature requests
- Documentation improvements

Use discussions when available for:

- Design questions
- Operational practices
- Workflow ideas

## Before Opening An Issue

- Review README.md and docs/README.md.
- Check docs/faq.md.
- Include logs, command output, and environment details.

## Suggested Issue Details

- Pipeline version
- Python version
- Command or endpoint used
- Minimal reproduction steps
- Expected result
- Actual result
